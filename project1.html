Project #1: Bare-Metal Self-Hosting with Raspberry Pi

GitHub Repo

Stack: Raspberry Pi 5 • Nginx • Bash • GitHub Actions (CI/CD) • HTML5/CSS3

Objective: Architect a full-stack web presence from first principles, bypassing managed hosting services to learn more about web infrastructure and deployment.

Infrastructure and Architecture

Rather than relying on abstracted cloud hosting, I serve this site from a Raspberry Pi 5 running in my home network. This "bare-metal" approach requires managing the OS layer directly. I interact with the server exclusively via SSH (headless), enforcing a command-line-first workflow that mirrors the environment of high-performance computing clusters used in my research.

Automated Deployment (CI/CD)

To ensure code reproducibility and streamline updates, I engineered a Continuous Deployment pipeline. I configured a self-hosted GitHub Actions runner directly on the Pi.

Now, when I push changes to the repository, the runner automatically:

  • Pulls the latest commit
  • Builds the static assets
  • Deploys them to the Nginx serving directory

This eliminates manual file transfers (SCP) and ensures the live site always reflects the `main` branch state. This was also good practice for Python packaging workflows (Project 2).

Security and Networking

Exposing a local machine to the open web requires security hardening. I utilize Nginx as a high-performance reverse proxy and secure the network using UFW (Uncomplicated Firewall) and Fail2Ban to automatically mitigate intrusion attempts. I also ban IPs with suspicious behavior, which is easy to audit with GoAccess, a real-time log analysis tool.
